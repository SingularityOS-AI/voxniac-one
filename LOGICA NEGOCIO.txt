How to turn a prompt into a Claude Skill
A Skill is just a set of instructions you save once so Claude reuses it on its own. You don't need to code. Here's the whole thing.
1. Turn on file creation.
In Claude, go to Settings → Capabilities and make sure "Code execution and file creation" is toggled on.
2. Open the Skills area.
Go to Settings → Capabilities → Skills (on some accounts it's under Customize → Skills in the left sidebar).
3. Create a new Skill.
Click the + button, then Create skill. You'll see a few options. "Create with Claude" is the easiest: paste in one of the prompts below and tell Claude "turn this into a Skill for me." It builds the file for you.
4. Or build the file yourself.
If you'd rather do it by hand, a Skill is one plain-text file called . It has two parts:
A short header (called frontmatter) with a name and a description. The description is the important bit: it's how Claude decides when to fire the Skill, so write it in plain "use this when..." language, max 200 characters.
The instructions underneath, which is just the prompt itself.
Here's what the top of the Offer Builder Skill would look like:
---name: Offer Builder
description: Use when the user wants to sharpen, rebuild, or pressure-test their offer, pricing, or guarantee to book more calls. Modeled on Hormozi's $100M Offers.
---
[paste the full prompt here]
5. Save it.
If you built the file by hand, put it in a folder, zip the folder, then upload the ZIP in the Skills area (the + → Create skill → upload option). If you used "Create with Claude," it saves automatically.
6. Use it.
From now on, when you tell Claude something like "help me fix my offer," it recognizes the job from your Skill's description and runs the framework without you pasting anything. Do this once per Skill and you've got all five living inside Claude permanently.
 Naming tip for the description field: describe the problem it solves, not the guru it's based on. "Use when rebuilding an offer or pricing" triggers far more reliably than "Hormozi Skill," because Claude matches it against what you're actually asking for.
Skill 1 — The Offer Builder
Modeled on: Alex Hormozi, $100M Offers
 What it does: Rebuilds your offer so the right prospects reply instead of scrolling past. When the offer is strong enough, it does most of the selling before anyone gets on a call.
The core idea Hormozi teaches: make your offer so good people feel stupid saying no. You get there by raising the dream outcome and the perceived likelihood of success, while lowering the time delay and the effort it takes to get there.

Skill 2 — The Acquisition Builder
Modeled on: Iman Gadzhi, agency client-acquisition playbook
 What it does: Restructures how you win clients so leads come in on repeat, instead of the random feast-or-famine bursts you can't predict or plan around.
The core idea Gadzhi teaches: pick a niche with money and real pain, sell the outcome not the deliverable ("sell the vacation, not the flight"), and lead with free value before you ever pitch.
The prompt
Skill 3 — The Attention Builder
Modeled on: Gary Vaynerchuk, Jab, Jab, Jab, Right Hook + "document, don't create"
 What it does: Rewires how you show up so your market already knows and trusts you before you ever reach out. Warm beats cold every time.
The core idea Gary teaches: give value relentlessly (the jabs) so you earn the right to ask (the right hook). Then stop inventing content and start documenting what you already do.
The prompt
Skill 4 — The Funnel Builder
Modeled on: Russell Brunson, DotCom Secrets, Expert Secrets, Traffic Secrets
 What it does: Maps your value ladder so every lead has a clear path from first touch to booked call, with no leaks in between. A website is a maze. A funnel is a guided path.
The core idea Brunson teaches: ascend people up a value ladder using Hook, Story, Offer at every step, and go find your customers where they already gather (the Dream 100).
The prompt
Skill 5 — The Traffic Builder
Modeled on: Neil Patel, data-driven SEO and content marketing
 What it does: Restructures how you pull in inbound so qualified prospects find you, instead of you chasing them. The channel compounds while you sleep.
The core idea Patel teaches: target what your buyer actually searches, build content clusters around it, distribute that content everywhere, then track the data and double down on what works.
